# 用户管理功能 - 快速测试指南

## 🚀 快速测试步骤

### 准备工作
1. 启动系统：
```bash
npm run dev
# 或
./scripts/startup/start-dev.ps1
```

2. 使用默认账户登录：
   - 用户名: `admin`
   - 密码: `admin123`

---

## ✅ 测试场景

### 测试1：创建新用户（手动设置密码）

1. 导航到 **系统设置** → **用户管理**
2. 点击"新增用户"按钮
3. 填写信息：
   - 用户名: `test_user1`
   - 初始密码: `Test123456!`
   - 角色: 运维
   - 账户状态: 启用
   - 勾选"强制首次登录修改密码"
4. 点击"创建用户"
5. 点击页面顶部"保存设置"
6. **验证**：用户出现在列表中

---

### 测试2：自动生成密码

1. 点击"新增用户"
2. 填写用户名: `test_user2`
3. 点击"🎲 自动生成强密码"
4. **验证**：
   - 密码自动填充
   - 密码强度显示为"强"（绿色）
   - 密码自动显示（可见）
5. 复制生成的密码（用于后续登录测试）
6. 完成创建并保存

---

### 测试3：密码强度指示器

1. 点击"新增用户"
2. 填写用户名: `test_user3`
3. 在密码框依次输入：
   - `123` → 应显示：无指示器
   - `12345678` → 应显示：弱（红色）
   - `Test1234` → 应显示：中（橙色）
   - `Test123!@#` → 应显示：强（绿色）
4. **验证**：强度指示器正确显示

---

### 测试4：删除用户确认

1. 在用户列表中选择一个测试用户
2. 点击"删除"按钮
3. **验证**：出现确认对话框
4. 点击"取消" → 用户未被删除
5. 再次点击"删除" → "确定删除"
6. **验证**：用户被删除

---

### 测试5：编辑用户信息

1. 点击某个用户的"编辑"按钮
2. 修改角色为"管理员"
3. 在"新密码"框输入新密码（可选）
4. 点击"保存修改"
5. 点击页面顶部"保存设置"
6. **验证**：修改已保存

---

### 测试6：使用新账户登录

1. 登出当前账户
2. 使用新创建的账户登录：
   - 用户名: `test_user1`
   - 密码: `Test123456!`
3. **验证**：
   - 能够成功登录
   - 如果勾选了"强制修改密码"，应该提示修改密码

---

### 测试7：修改密码功能

1. 以任意账户登录
2. 点击右上角头像 → "修改密码"
3. 填写：
   - 当前密码: （当前账户密码）
   - 新密码: `NewPass123!`
   - 确认密码: `NewPass123!`
4. 点击"确认修改"
5. **验证**：密码修改成功提示
6. 登出后使用新密码登录测试

---

### 测试8：表单验证

测试以下验证是否正常工作：

**用户名验证**：
- [ ] 空用户名被拒绝
- [ ] 少于2字符被拒绝
- [ ] 特殊字符（如`@#$`）被拒绝
- [ ] 重复用户名被拒绝

**密码验证**：
- [ ] 新增时空密码被拒绝
- [ ] 少于8位被拒绝
- [ ] 编辑时空密码表示不修改

---

### 测试9：审计日志检查

1. 执行几次用户创建/修改操作
2. 检查日志文件：
```bash
# 查看API日志
cat logs/api.log | grep "用户管理操作"

# 或在Windows
Get-Content logs/api.log | Select-String "用户管理操作"
```
3. **验证**：日志包含操作记录

---

### 测试10：密码加密验证

1. 创建一个测试用户（密码: `Test123456!`）
2. 保存后查看配置文件：
```bash
cat ../configs/system-config.json
```
3. **验证**：
   - 密码字段包含`$2a$10$`开头的哈希值
   - 不是明文存储

## 🔍 故障排查

### 问题：无法登录新账户
**解决**：
1. 检查是否点击了"保存设置"
2. 查看项目根目录的`configs/system-config.json`是否有该用户
3. 检查密码是否为加密格式（`$2a$10$...`）

### 问题：密码强度不显示
**解决**：
1. 确保密码长度至少1位
2. 检查浏览器控制台是否有错误
3. 刷新页面重试

### 问题：删除没有确认框
**解决**：
1. 检查前端代码是否正确部署
2. 清除浏览器缓存
3. 重新构建前端：`cd main-portal && npm run build`
