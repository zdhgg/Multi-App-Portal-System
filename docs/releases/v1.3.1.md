# v1.3.1 - UTF-8 BOM 配置兼容性修复补丁

发布日期：2026-03-30

## Highlights

- 修复 `system-config.json` 以 UTF-8 BOM 保存时，后端多个入口可能出现解析失败的问题。
- 将认证、系统设置、文件系统路径白名单、路径安全和自动备份调度统一切换到 BOM 安全的配置读取链路。
- 补充 BOM 兼容回归测试，并同步前后端版本、脚本模板和发布文档到 `1.3.1`。

## Included Work

- `detection-api/src/utils/systemConfigPath.ts` 新增 BOM 剥离、统一配置解析和同步/异步文件读取封装。
- `detection-api/src/api/controllers/AuthController.ts`、`detection-api/src/controllers/SystemSettingsController.ts` 改为通过统一解析入口读取系统配置。
- `detection-api/src/routes/filesystem.ts` 与 `detection-api/src/core/security/PathSecurityManager.ts` 改为共享 BOM 安全的路径白名单读取逻辑。
- `detection-api/src/services/backupScheduler.ts` 修复自动备份调度器在 BOM 配置文件场景下的计划加载问题。
- `detection-api/src/utils/__tests__/systemConfigPath.test.ts` 与 `detection-api/src/services/__tests__/backupScheduler.test.ts` 新增 BOM 兼容回归测试。
- 同步根项目、前端、后端、系统配置、启动脚本和主要发布文档中的版本标识到 `1.3.1`。

## Compatibility

- 本次为语义化版本 `1.3.1` 的补丁发布。
- 未引入已知的破坏性接口、数据库结构或配置格式变更。
- 兼容现有 `system-config.json` 内容，仅增强对 UTF-8 BOM 文件格式的容错能力。
