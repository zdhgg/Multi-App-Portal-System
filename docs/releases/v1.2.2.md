# v1.2.2 - PM2 进程识别与停止修复补丁

发布日期：2026-03-26

## Highlights

- 新增按应用 ID 停止 PM2 进程的接口，解决应用名与 PM2 进程名不一致时难以正确停止的问题。
- 收紧 PM2 目录匹配规则，仅接受应用目录及其子目录，避免共享工作区父目录造成误匹配。
- Windows 原生进程停止改为优先终止完整进程树，提升端口释放与异常退出处理稳定性。

## Included Work

- `detection-api/src/routes/pm2.ts` 新增 `/api/pm2/apps/:appId/stop` 路由，并统一停止后的应用状态同步逻辑。
- `detection-api/src/services/pm2StateSyncService.ts` 与 `main-portal/src/views/Management.vue` 更新 PM2 匹配策略，支持名称与目录双重识别。
- `main-portal/src/utils/pm2ProcessMatching.ts` 与对应测试新增共享父目录误匹配防护。
- `detection-api/src/core/SimpleServices.ts` 优化 Windows 子进程树终止流程，并保留信号兜底。
- 同步根项目、前端、后端、配置模板、脚本头部和发布文档中的版本标识到 `1.2.2`。

## Compatibility

- 本次为语义化版本 `1.2.2` 的补丁发布。
- 未引入已知的破坏性接口或配置格式变更。
