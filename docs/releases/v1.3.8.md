# v1.3.8 - 目录选择回填与全栈工作区识别增强补丁

发布日期：2026-04-27

## Highlights

- 修复“系统设置 > 路径访问”中“选择目录并添加”在存在空白输入行时看起来没有成功添加的问题，并统一目录回填与去重行为。
- 路径访问场景的目录选择新增按场景放宽的返回路径校验，避免出现“手动输入可以，目录选择失败”的不一致体验。
- 全栈识别、应用配置验证和配置服务统一增强对 `frontend/client/web/ui/portal`、`backend/server/api/services` 以及 `apps/*` 工作区结构的支持。
- PM2 状态同步补充本地端口监听探测，external-exe 工作目录变更后会自动重映射 `build_script`，运行态与生命周期修复更完整。

## Included Work

- `main-portal/src/components/settings/PathAccessSettingsPanel.vue`、`main-portal/src/utils/pathAccessList.ts` 修复路径访问“选择目录并添加”的回填逻辑，优先复用空白行并阻止重复路径。
- `main-portal/src/utils/directoryPicker.ts`、`main-portal/src/components/FolderBrowser.vue`、`main-portal/src/services/filesystemApi.ts` 为目录选择链路新增 `validateSelectedPath` 场景开关，并统一本机/服务器目录模式提示。
- `main-portal/src/components/ManualAddApp.vue`、`main-portal/src/components/detection/PathSelector.vue`、`main-portal/src/components/settings/BackupRestorePanel.vue`、`main-portal/src/views/Detection.vue` 统一目录选择入口文案，明确服务器目录浏览上下文。
- `detection-api/src/api/controllers/FilesystemController.ts` 支持按调用场景关闭目录选择结果的即时存在性校验，避免路径白名单编辑场景被误拦截。
- `detection-api/src/core/ApplicationService.ts`、`detection-api/src/services/enhancedFullStackDetector.ts` 增强 monorepo 与工作区全栈识别，支持 `package.json workspaces` 与 `pnpm-workspace.yaml`。
- `detection-api/src/services/AppConfigValidator.ts`、`detection-api/src/services/appConfigurationService.ts` 扩展前后端候选目录，覆盖 `apps/web`、`apps/api` 等常见工作区布局。
- `detection-api/src/services/pm2StateSyncService.ts` 改进运行态识别，结合端口冲突结果和本地监听探测判断手动启动应用的实际状态。
- `detection-api/src/core/__tests__/ApplicationService.lifecycle.test.ts`、`detection-api/src/services/__tests__/enhancedFullStackDetector.test.ts`、`main-portal/src/tests/path-access-list.test.ts` 新增对应回归测试。

## Compatibility

- 本次为语义化版本 `1.3.8` 的补丁发布。
- 未引入已知的破坏性接口、数据库结构或配置文件格式变更。
- 远程访问下的目录选择仍然针对部署服务器路径，不会直接返回当前浏览器所在客户端机器的本地目录。
