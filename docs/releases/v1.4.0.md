# v1.4.0 - 全栈端口治理与门户图标体验更新

发布日期：2026-05-24

## Highlights

- 全栈应用支持按前端/后端端口范围自动分配同偏移端口组，降低前后端端口错位和手动调整成本。
- Windows 端口释放链路改为精确匹配监听端口并终止进程树，减少相邻端口误匹配和子进程残留。
- 应用配置页恢复主端口、次要端口和协议编辑，并同步写回应用网络配置。
- 门户图标体系迁移到统一图标选择组件，支持 Element Plus 图标、自定义文本/Emoji 和图片链接。

## Included Work

- `detection-api/src/core/ApplicationService.ts` 新增全栈端口配对分配、指定端口分配 fallback、端口更新校验和停止后释放验收。
- `detection-api/src/core/NetworkService.ts` 新增 `allocateSpecificPort`、`allocatePortPair`，并在释放端口后校验系统监听状态。
- `detection-api/src/services/PortManagementService.ts`、`detection-api/src/utils/portCleaner.ts` 强化 Windows 端口检测和进程树清理。
- `detection-api/src/routes/appConfiguration.ts` 通过应用服务获取完整应用信息，补齐端口配置所需的目录、技术栈和全栈上下文。
- `main-portal/src/components/AppConfiguration.vue` 支持编辑主端口、次要端口和协议，并同步到应用更新接口。
- `main-portal/src/components/common/IconSelect.vue` 新增统一图标选择器，管理页、手动添加和外观编辑共用同一套图标输入体验。
- 门户首页、应用卡片、详情弹窗、管理页、系统设置和侧栏调整为更克制的产品界面样式。

## Compatibility

- 本次为语义化版本 `1.4.0` 的小版本发布。
- 应用配置 API 的推荐路径为 `/app-configurations`；后端保留 `/app-configuration` 兼容别名，旧前端仍可请求原路径。
- 端口配置数据结构继续使用 `network.primaryPort`、`network.secondaryPorts` 和 `network.protocol`。
- 停止应用时，如果端口仍被占用，现在会返回 `STOP_INCOMPLETE`，调用方需要按错误提示处理残留进程或权限问题。
