# v1.3.9 - 应用运行时间显示修复补丁

发布日期：2026-04-27

## Highlights

- 修复门户首页与应用详情中的“运行时间”长期显示为“尚未启动”的问题，运行中应用现在会展示真实存活时长。
- 公共应用接口不再把 `uptime` 固定返回为 `0`；现在会优先读取 PM2 uptime，并在直启/开发模式下回退到进程管理器中的 `startedAt`。
- 前端展示链路不再把 `uptime=0` 的在线应用误判为未启动，刚启动或同步中的应用会按实际状态稳定展示。

## Included Work

- `detection-api/src/api/controllers/PublicController.ts` 为公共应用数据增加真实 uptime 解析，覆盖 PM2 托管应用与直接启动应用。
- `detection-api/src/core/ServiceContainer.ts` 将 `processManager` 注入到 `PublicController`，用于直启应用的运行时间回填。
- `main-portal/src/components/portal/AppCard.vue`、`main-portal/src/components/portal/AppDetailDialog.vue` 调整运行时间文案判定，避免把在线应用误显示为“尚未启动”。
- `detection-api/src/api/controllers/__tests__/PublicController.test.ts` 新增回归测试，覆盖 PM2 uptime 时间戳换算与直启应用 `startedAt` 计算。

## Compatibility

- 本次为语义化版本 `1.3.9` 的补丁发布。
- 未引入已知的破坏性接口、数据库结构或配置文件格式变更。
- 门户运行时间仍依赖当前进程状态；离线应用继续显示“尚未启动”。
