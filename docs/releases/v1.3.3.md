# v1.3.3 - 目录选择兼容性与批量导入体验优化补丁

发布日期：2026-04-07

## Highlights

- 新增网页式服务器目录浏览器，局域网远程访问时不再强依赖后端原生目录选择窗口。
- 手动添加、批量导入、备份路径和路径白名单配置统一复用新的最佳努力目录选择链路。
- 检测页进一步收口为低频批量导入工具，默认单应用入口统一回到“应用管理 > 添加应用”。
- 应用启动遇到端口冲突时，管理页会直接给出跳转到端口管理的提示，并聚焦冲突端口。

## Included Work

- `detection-api/src/api/controllers/FilesystemController.ts` 新增默认目录可读性校验，并开放 `/filesystem/home`、`/filesystem/browse`、`/filesystem/validate` 支撑网页目录浏览。
- `main-portal/src/utils/directoryPicker.ts` 新增网页目录浏览回退逻辑，统一替换手动添加、批量导入、备份设置和路径访问配置中的目录选择入口。
- `main-portal/src/views/Detection.vue`、`main-portal/src/components/detection/ProgressMonitor.vue` 将检测页收口为低频批量导入流程，并强调单应用默认走手动添加。
- `main-portal/src/views/Management.vue`、`main-portal/src/views/PortManagement.vue`、`main-portal/src/components/PortManager.vue` 增强端口冲突提示与聚焦定位。
- `main-portal/src/stores/auth.ts`、`main-portal/src/utils/jwt.ts`、`main-portal/src/components/auth/LoginDialog.vue` 修复 base64url JWT 恢复和“记住我”用户名回填问题。
- `detection-api/src/core/SimpleServices.ts` 保留版本化 API 前缀，避免 `/api/v1` 等路径在 Vite 代理场景下被误改写。

## Compatibility

- 本次为语义化版本 `1.3.3` 的补丁发布。
- 未引入已知的破坏性接口、数据库结构或配置格式变更。
- 远程访问场景下，目录选择会落到服务器目录浏览器；如需选择客户端本机目录，仍需在服务器本机打开系统页面。
