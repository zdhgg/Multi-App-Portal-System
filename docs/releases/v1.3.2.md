# v1.3.2 - 启动校验与运维诊断增强补丁

发布日期：2026-03-31

## Highlights

- `start-production.ps1` 升级为校验式启动流程，要求 PM2 在线、监听 PID 匹配且 `/health` 连续通过后才视为启动成功。
- `Start-Portal.bat` 新增防火墙与 PM2 开机自启状态检测，重启与构建后重启统一复用新的校验式启动入口。
- `detection-api` 在 PM2 场景下会将启动期日志镜像到 stdout/stderr，并补充启动前后关键日志，便于排查秒退和空日志问题。
- 新增内网环境回归测试方案文档，覆盖离线拷贝部署、端口冲突、PM2 权限异常、配置异常和重启恢复场景。

## Included Work

- `start-production.ps1` 新增启动就绪等待、健康检查重试、端口归属校验、失败时日志摘录和环境提醒输出。
- `Start-Portal.bat` 新增防火墙状态探测、开机自启状态展示，并将重启/构建后重启统一切换到 `run_verified_start`。
- `detection-api/pm2-entry.mjs` 新增 PM2 启动前后日志，启动失败时会显式输出异常。
- `detection-api/src/utils/logger.ts` 在 PM2 或显式开启 `LOG_TO_STDOUT` 时增加控制台输出，避免 PM2 日志为空。
- `docs/project/内网环境回归测试方案-2026-03-31.md` 新增离线内网部署回归测试方案。
- 同步根项目、前端、后端、系统配置、脚本模板和主要发布文档中的版本标识到 `1.3.2`。

## Compatibility

- 本次为语义化版本 `1.3.2` 的补丁发布。
- 未引入已知的破坏性接口、数据库结构或配置格式变更。
- 现有部署方式保持不变，但启动成功判定更加严格，能更早暴露端口冲突和服务秒退问题。
