## 🎬 真实使用场景

### 场景1：端口被占用

**问题：** 启动应用时提示端口占用

**传统做法：**
1. 打开PowerShell
2. 运行 `netstat -ano | findstr :3041`
3. 找到PID
4. 运行 `taskkill /PID xxx /F`
5. 回到门户重新启动

**现在：**
1. 点击"一键智能诊断"
2. 点击"自动修复"
3. 完成！

### 场景2：依赖未安装

**问题：** 应用启动失败，提示模块未找到

**传统做法：**
1. 找到应用目录
2. 打开PowerShell
3. cd 到应用目录
4. 运行 npm install
5. 等待安装完成
6. 回到门户重新启动

**现在：**
1. 点击"一键智能诊断"
2. 点击"自动修复"
3. 等待安装
4. 完成！

## ❓ 常见问题

### Q: 诊断会修改我的应用吗？
**A:** 不会。诊断只是读取和检查，不会修改任何东西。只有点击"自动修复"才会执行修复操作。

### Q: 自动修复安全吗？
**A:** 是的。修复前会显示确认对话框，并且只修复明确知道如何修复的问题（端口释放、依赖安装）。

### Q: 所有问题都能自动修复吗？
**A:** 不是。只有端口冲突和依赖缺失可以自动修复。其他问题会提供详细的手动修复指南。

### Q: 诊断需要网络吗？
**A:** 诊断本身不需要网络。但如果自动修复需要安装依赖（npm install），则需要网络连接。

### Q: 修复失败了怎么办？
**A:** 诊断结果中有详细的修复命令，您可以复制到PowerShell手动执行。或查看《PM2错误快速解决方案.md》。
