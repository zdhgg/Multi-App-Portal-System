# 快速修复 PM2 问题

## 🎯 一句话总结

**PM2 已启用但无法连接？下载修复脚本，右键"以管理员身份运行"即可！**

## 📋 问题说明

### 您看到的诊断结果

- ✅ **操作系统：win32 - Windows** （正确）
- ✅ **PM2 状态：已启用** （正确，环境变量已设置）
- ❌ **问题：PM2_CONNECTION_ERROR** （PM2 守护进程无法连接）

### 这是什么意思？

虽然您的配置文件中启用了 PM2（`PM2_ENABLED=1`），但 PM2 的守护进程出现了连接问题。这通常是由于：

1. **权限问题**（最常见）
   - PM2 守护进程需要管理员权限
   - 后端服务运行在普通用户权限下

2. **守护进程崩溃**
   - PM2 守护进程意外终止
   - 需要重新启动

3. **配置冲突**
   - 多个 PM2 实例冲突
   - 需要清理并重置

## 🚀 最简单的解决方案

### 步骤 1：下载修复脚本

1. 在 PM2 修复对话框中，**向下滚动**
2. 找到 **"方案 1：使用修复脚本（推荐）"**
3. 点击 **"下载修复脚本"** 按钮
4. 脚本会自动下载到您的下载文件夹

### 步骤 2：运行修复脚本

1. 打开 **下载** 文件夹
2. 找到 `fix-pm2-admin.bat` 文件
3. **右键点击** 该文件
4. 选择 **"以管理员身份运行"**
5. 在 UAC 提示中点击 **"是"**

### 步骤 3：等待完成

脚本会自动执行以下操作：

```
✅ 检查管理员权限
✅ 终止 PM2 守护进程 (pm2 kill)
✅ 重新初始化 PM2 (pm2 ping)
✅ 验证 PM2 状态 (pm2 list)
```

### 步骤 4：重新尝试

1. 返回应用管理界面
2. 点击 **"重新诊断"** 或直接尝试启动应用
3. 问题应该已经解决！

## ❓ 常见问题

### Q: 为什么需要管理员权限？

**A:** Windows 的安全机制不允许程序自动提升权限。PM2 使用命名管道进行进程间通信，某些操作需要管理员权限才能访问。

### Q: 自动修复为什么会失败？

**A:** 后端服务运行在普通用户权限下，无法执行需要管理员权限的命令。这是 Windows 的安全设计，防止恶意软件自动获取高权限。

### Q: 修复后还是失败怎么办？

**A:** 请检查：

1. **PM2 是否正确安装**
   ```powershell
   pm2 --version
   ```

2. **查看详细错误信息**
   - 在对话框中查看修复步骤的详细输出
   - 查看应用的日志文件

3. **尝试直接启动模式**
   - 在对话框中点击"直接启动应用"
   - 绕过 PM2，使用 Node.js 直接启动

### Q: 每次都要手动修复太麻烦？

**A:** 有两个长期方案：

1. **以管理员权限运行后端服务**（推荐）
   - 使用 `scripts\startup\start-backend-admin.ps1` 脚本
   - 自动修复功能可以正常工作

2. **使用直接启动模式**
   - 在应用配置中设置启动方式为"直接启动"
   - 不需要 PM2，但失去进程管理功能
