# PM2 修复脚本使用说明

## 📦 两个版本的脚本

我们提供了两个版本的 PM2 修复脚本，您可以根据自己的情况选择使用：

### 1. 批处理脚本 (fix-pm2-admin.bat) ⭐ 推荐

**适用场景：**
- ✅ 大多数 Windows 用户
- ✅ 不熟悉 PowerShell 的用户
- ✅ 需要最简单操作的用户

**优点：**
- 🎯 **最简单**：双击或右键"以管理员身份运行"即可
- 🔧 **兼容性好**：所有 Windows 版本都支持
- 🚀 **无需额外配置**：不需要修改执行策略

**使用方法：**
1. 下载 `fix-pm2-admin.bat`
2. 右键点击文件
3. 选择 **"以管理员身份运行"**
4. 等待完成

**特点：**
- 自动检测管理员权限
- 自动查找 PM2 安装位置
- 自动添加常见的 npm 路径到 PATH
- 如果找不到 PM2，会提供详细的错误信息

---

### 2. PowerShell 脚本 (fix-pm2-admin.ps1) 🔥 高级

**适用场景：**
- ✅ 熟悉 PowerShell 的用户
- ✅ 需要更详细输出的用户
- ✅ 批处理脚本无法解决问题时

**优点：**
- 💪 **功能更强**：更智能的路径查找
- 🎨 **输出更美观**：彩色输出，更易读
- 🔍 **错误处理更好**：更详细的错误信息

**使用方法：**

#### 方法 1：右键运行（推荐）
1. 下载 `fix-pm2-admin.ps1`
2. 右键点击文件
3. 选择 **"使用 PowerShell 运行"**
4. 在 UAC 提示中点击"是"

#### 方法 2：在 PowerShell 中运行
1. 以管理员身份打开 PowerShell
   - 按 `Win + X`
   - 选择 "Windows PowerShell (管理员)"
2. 导航到脚本所在目录
   ```powershell
   cd "C:\Users\YourName\Downloads"
   ```
3. 运行脚本
   ```powershell
   .\fix-pm2-admin.ps1
   ```

**注意事项：**
- 如果遇到"无法加载，因为在此系统上禁止运行脚本"错误：
  ```powershell
  Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
  ```
- 运行完脚本后，可以恢复原来的策略：
  ```powershell
  Set-ExecutionPolicy -ExecutionPolicy Restricted -Scope CurrentUser
  ```

**特点：**
- 使用 `Get-Command` 智能查找 PM2
- 彩色输出（绿色=成功，红色=错误，黄色=警告）
- 更详细的错误信息和调试输出
- 更好的异常处理

### 选择批处理脚本 (.bat) 如果：
- ✅ 您不熟悉命令行工具
- ✅ 您想要最简单的解决方案
- ✅ 您不想修改 PowerShell 执行策略

### 选择 PowerShell 脚本 (.ps1) 如果：
- ✅ 您熟悉 PowerShell
- ✅ 批处理脚本无法解决您的问题
- ✅ 您需要更详细的诊断信息
- ✅ 您想要更美观的输出

## 🔧 两个脚本都做了什么？

两个脚本执行相同的修复步骤，只是实现方式不同：

### 步骤 1：检查管理员权限
- 确认脚本以管理员身份运行
- 如果没有权限，提示用户并退出

### 步骤 2：查找 PM2 命令
- 检查 PM2 是否在系统 PATH 中
- 如果没找到，尝试添加常见的 npm 安装路径：
  - `%APPDATA%\npm`
  - `%ProgramFiles%\nodejs`
  - `%ProgramFiles(x86)\nodejs`
  - `%USERPROFILE%\AppData\Roaming\npm`

### 步骤 3：终止 PM2 守护进程
```bash
pm2 kill
```
- 终止当前运行的 PM2 守护进程
- 清理可能损坏的进程状态

### 步骤 4：重新初始化 PM2
```bash
pm2 ping
```
- 启动新的 PM2 守护进程
- 验证 PM2 是否正常工作

### 步骤 5：验证状态
```bash
pm2 list
```
- 显示当前 PM2 管理的所有进程
- 确认 PM2 已正常运行

## ❓ 常见问题

### Q1: 为什么需要管理员权限？

**A:** PM2 使用 Windows 命名管道进行进程间通信，某些操作需要管理员权限才能访问。这是 Windows 的安全机制。

### Q2: 脚本报错"找不到 PM2 命令"怎么办？

**A:** 这说明 PM2 可能没有正确安装，或者安装路径不在系统 PATH 中。

**解决方案：**
1. 在普通 PowerShell 中重新安装 PM2：
   ```powershell
   npm install -g pm2
   ```
2. 检查 PM2 是否安装成功：
   ```powershell
   pm2 --version
   ```
3. 如果还是不行，尝试使用 PowerShell 版本的脚本

### Q3: 批处理脚本和 PowerShell 脚本有什么区别？

**A:** 主要区别：

| 特性 | 批处理脚本 (.bat) | PowerShell 脚本 (.ps1) |
|------|------------------|----------------------|
| 易用性 | ⭐⭐⭐⭐⭐ 最简单 | ⭐⭐⭐⭐ 需要了解 PS |
| 兼容性 | ⭐⭐⭐⭐⭐ 所有 Windows | ⭐⭐⭐⭐ Windows 7+ |
| 功能性 | ⭐⭐⭐⭐ 基本功能 | ⭐⭐⭐⭐⭐ 功能更强 |
| 输出美观度 | ⭐⭐⭐ 纯文本 | ⭐⭐⭐⭐⭐ 彩色输出 |
| 错误处理 | ⭐⭐⭐⭐ 良好 | ⭐⭐⭐⭐⭐ 优秀 |

### Q4: PowerShell 脚本提示"无法加载，因为在此系统上禁止运行脚本"？

**A:** 这是 PowerShell 的执行策略限制。

**临时解决方案：**
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

**或者直接使用批处理脚本**，不需要修改执行策略。

### Q5: 脚本执行后还是无法启动应用？

**A:** 请检查：

1. **PM2 是否正常运行**
   ```powershell
   pm2 list
   ```

2. **查看应用的详细日志**
   - 在应用管理界面查看日志
   - 或使用 `pm2 logs <app-name>`

3. **尝试直接启动模式**
   - 在 PM2 修复对话框中选择"直接启动应用"
   - 绕过 PM2，使用 Node.js 直接启动

4. **检查应用配置**
   - 确认应用的启动脚本路径正确
   - 确认应用的端口没有被占用
